# Security Policy

## Supported Versions

We actively support and provide security updates for the following versions of Nextora Studio:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of Nextora Studio seriously. If you believe you have found a security vulnerability, please do not report it publicly via GitHub issues. Instead, please report it directly through our security disclosure channel:

- **Email**: security@nextora.studio
- **Response Time**: You will receive an acknowledgment of your report within 24–48 hours, along with a tracking ID and details on the next steps.
- **Updates**: We will keep you informed of our progress as we investigate and work on a resolution.

Please provide a detailed description of the vulnerability, including:
1. Steps to reproduce the issue.
2. A proof-of-concept (PoC) if available.
3. The potential impact of the vulnerability.

## Disclosure Policy

We follow standard responsible disclosure practices:
- We ask you to give us reasonable time to investigate and patch the issue before making any details public.
- We will coordinate public disclosure dates with the reporter where appropriate.
- Credit will be given in our changelogs and release notes once the vulnerability is successfully mitigated.

Thank you for helping keep Nextora Studio secure!
